Prerequisite :

The requirements for this tutorial are:
  1. Java 1.8
  2. NetBeans 8.2
  3. Jeddict 4.5
  4. Java EE 8 Server (e.g Payara 5.0.0.173 Snapshot)

Setting Up the Project :

To create an IDE project:
  • Start NetBeans IDE.
  • In the IDE, choose File > New Project(Ctrl-Shift-N) from the main menu, as shown in the figure below :
  • In the New Project wizard, expand the Maven category and select Web Application as shown in the figure below. Then click Next.
  • Enter ShoppingSystem in the Project name text box and specify the Project Location to any directory on your computer and click next (as shown in the figure below):
  • Select the version of Java EE you want to use with your application and the server to which you want to deploy your application.
  • Click Finish.

Create the JPA Diagram :

  1. Goto File menu > New File > Persistence category.
  2. From the Persistence , select JPA Diagram and click Next.
  3. Type CartModel for the diagram name and Browse the location in project.
  4. Click Finish.
  5. When you click Finish, the IDE creates the JPA Diagram and opens the diagram in the designer window.

Create the Entities :

  1. Drag & Drop the Entity from the Palette window to designer.
  2. Double click on the Entity and rename to Customer.

Add properties :

  1. To create @Basic attribute, click on the Customer Entity , then click on the expand button > basic property.
  2. Same as above create some other entities and properties (for more information click here).

Front-end properties :

  1. Set entity display reference attribute : Click on attribute > Properties > UI properties > Entity Display Reference :
  2. Exclude attribute in UI : Click on attribute > Properties > UI properties > Include in UI :

Generate the entity classes :

  • Right click on the diagram > Generate Source Code or (ctrl + G)

  • Business Service :

    Click on the Business Service > select Session Facade. In the following screen, you can configure the SessionBean package and class name pattern for the EJB facade that will be created for your application:

  • REST Controller :

    Click on the Controller > select REST
    Implement the Controller:
    In the following screen, you can configure the Controller package and class name pattern for the REST api that will be created for your application:
    Configure the application:
    To create our REST application we also need a JAX-RS Application class: The subclass of javax.ws.rs.core.Application can be used to define additional JAX-RS components. In the following screen, you can configuration @ApplicationPath (that define the base path for our application) and class name.

Limitation

Data Types

JPA Modeler, DB Modeler or JSON Modeler supports all datatype based on the corresponding specification like JPA or JSONB datatype support;
But front-end templates supports only the following listed data types :
  • String
  • boolean, Boolean
  • int, Integer
  • long, Long
  • float, Float
  • double, Double
  • java.math.BigDecimal
  • java.time.LocalDate
  • java.time.Instant
  • java.time.ZonedDateTime
  • Enum

Other Limitation

Front-end templates also does not support the code generation for :
  • Embeddable
  • @ElmentCollection
  • Composite Primary key
